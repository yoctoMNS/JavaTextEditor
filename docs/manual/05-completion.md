# 5. コード補完

INSERTモードで、識別子の入力に合わせてコード補完ポップアップを利用できます。2つのトリガーキーがあり、参照するインデックスが異なります。

## `Ctrl+Space`（マージ補完）

`WordIndex`（作業ディレクトリ配下のファイル + 現在バッファの単語。フィールド・メソッド・ローカル変数・定数を含む）を最優先で表示し、埋まらなかった残り枠に `CompletionIndex`（JDKクラス名のみ）の候補を重複除去のうえ追加します。

## `Alt+/`（単語補完）

`WordIndex` のみを使う単語補完です（Vimの `i_CTRL-N` 相当）。JDKクラス名は候補に含まれません。

> 当初は `Ctrl+N` に割り当てる想定でしたが、INSERTモードの `Ctrl+N` は既にEmacs式「カーソル下移動」に確定済みのため、`Alt+/` が割り当てられています（`Ctrl+N` のカーソル下移動は維持されています）。

## 自動トリガー

INSERTモードで1文字入力するたびに、現在アクティブな補完セッションのモード（マージ/単語のみ）を維持したまま自動的に再クエリされ、候補があれば自動的にポップアップが表示されます。明示的にキーを押さなくても入力中に候補が出る「入力しながら補完」の挙動です。

## 候補の並び順

現在編集中バッファ内の単語は、Vimの `'complete'`（`i_CTRL-N` 既定値）に倣い、**カーソル位置からの近接順**で並びます。カーソル以降（ファイル末尾方向）の出現を先に、末尾に達したら先頭へ折り返してカーソル手前までを検索し、重複語は最初に見つかった出現（＝カーソルに近い方）だけを残します。他ファイル（ディスク索引）側の候補はアルファベット順のままです。カーソル位置そのものの「今まさに入力中の未確定なプレフィックス」は候補から除外されます。

`Ctrl+Space` 使用時、`CompletionIndex`（JDKクラス名）側の候補はスコアリング順で、`WordIndex` の近接順候補の後ろに追加されます。

## ポップアップ操作

| キー | 動作 |
|---|---|
| `↑` / `↓` | 候補を選択 |
| `Tab` / `Enter` | 選択中の候補を確定挿入 |
| `Escape` | ポップアップを閉じる |

カーソル移動・改行・単語削除などの操作でもポップアップは自動的に閉じます。

## インデックスの構築

- `WordIndex`・`CompletionIndex` はいずれもエディタ起動時に1回だけバックグラウンド（仮想スレッド）で構築されます。`:cd` で作業ディレクトリを変更しても再構築はされません
- ビルド中はCtrl+Space/Alt+/を押しても「補完インデックス構築中...」のメッセージが表示されるだけで、ビルド完了後に通常どおり利用できます
- `.git`/`build`/`target`/`node_modules` 等の慣例的なディレクトリと、2MBを超えるファイル・既知のテキスト拡張子以外のファイルはインデックス対象から除外されます
