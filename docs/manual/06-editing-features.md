# 6. 編集支援機能

## Getter/Setter自動生成

カーソル行のフィールド宣言（例: `private int hp;`）を解析して、getter/setterをクラス末尾の `}` 直前に自動挿入します。

| キー | 動作 |
|---|---|
| `Space+g+g` | getterを生成 |
| `Space+g+s` | setterを生成 |
| `Space+g+d` | getterとsetterを両方生成 |

- `boolean` 型のフィールドは `is` プレフィックスのgetterになります（Java Beans規約）
- 初期化式（`= ...`）付きのフィールド宣言や、識別子として不正なフィールド名は生成対象外です
- インデントスタイルはバッファ内の既存コードから自動検出されます（検出できない場合はスペース4つにフォールバック）

## インデント（VISUALモードの `>` / `<`）

VISUAL/VISUAL LINE/VISUAL BLOCKモードで選択した行を `>`（インデント）/`<`（デデント）できます。数字プレフィックス（例: `3>`）で段数を指定可能です。

- `shiftwidth`（既定4）単位でインデント幅を増減します
- `shiftround`（既定false）を有効にすると、現在のインデント幅を `shiftwidth` の倍数に丸めてから加減算します
- `expandtab`（既定true）によりインデントは常にスペースで構成されます
- 空白のみの行への右シフトは何もしません（Vimと同じ挙動）。左シフトは幅0でクランプされます

これらの設定（`shiftwidth`/`tabstop`/`expandtab`/`shiftround`）は現状 `IndentSettings` の固定デフォルト値であり、`:set` 相当のユーザー変更コマンドはまだありません。

## 括弧対応ジャンプ（`%`）

NORMAL/VISUAL/VISUAL LINE/VISUAL BLOCKモードで `%` を押すと、カーソル位置の括弧（`()` `[]` `{}`）に対応する括弧へジャンプします。ネストした括弧にも対応していますが、**カーソルが括弧の文字の上にある場合のみ**動作します（Vimのように行内の直近の括弧を自動的に探す挙動ではありません）。

## 自動インデント・括弧まわりの入力支援

| 動作 | 説明 |
|---|---|
| INSERTモードでの改行 | 現在行の先頭空白を新しい行にコピー。カーソル直前の行が `{` で終わっていればさらに1段インデントを追加 |
| `}` の入力 | 現在行がインデントのみの場合、1段分デデントしてから `}` を挿入 |
| `Tab`（閉じ括弧の直前） | `)` `]` `}` `"` `'` `>` の直前でTabを押すと、スペース挿入せずカーソルを括弧の外側へスキップ |

## マクロ（`q` 記録 / `@` 再生）

Vim互換のキーストローク記録・再生機能です。NORMALモードのキー操作だけでなく、INSERTモードでの文字入力やモード遷移（`i`/`ESC`など）をまたいで記録できます。

| キー | 動作 |
|---|---|
| `q{a-z}` | 指定したレジスタへマクロ記録を開始（小文字は新規上書き） |
| `q{A-Z}` | 対応する小文字レジスタの**既存内容へ追記**しながら記録を開始 |
| `q`（記録中） | 記録を終了 |
| `@{a-z}` | 指定したレジスタのマクロを1回再生 |
| `@@` | 直前に `@` で再生したレジスタをもう一度再生 |

- 記録中に打ったキーは、実際にその場で実行されつつ同時に記録されます（Vimと同じ）。記録し終えるまでの操作もバッファに反映されます。
- 記録中に別のレジスタを `@` で呼び出しても、その中身が展開されて記録されることはなく、「`@` と呼び出し先のレジスタ名」の2キーだけが記録されます。呼び出し先マクロの中身を後で編集しても、記録済みの呼び出し側マクロは自動的に追従します。
- 未記録のレジスタへ `@` を実行した場合や、`q`/`@` に `a`-`z`/`A`-`Z` 以外の文字を指定した場合は、ステータス行にエラーメッセージが表示されるだけでバッファは変化しません。
- `3@a` のようなカウント指定付きの繰り返し再生には対応していません（本エディタのNORMALモードには汎用的な数字カウントの前置き機構がそもそも存在しないため）。同じマクロを複数回実行したい場合は `@a` や `@@` を必要な回数だけ押してください。

## 行入れ替え

| キー | 動作 |
|---|---|
| `Alt+J` | 現在行と次行を入れ替え |
| `Alt+K` | 現在行と前行を入れ替え |

## ペイン分割・ナビゲーション

| キー/コマンド | 動作 |
|---|---|
| `sv` | 垂直分割（左右にペインを分割） |
| `ss` | 水平分割（上下にペインを分割） |
| `:sp` / `:split` | 垂直分割（コマンド版） |
| `:vs` / `:vsplit` / `:vsp` | 水平分割（コマンド版） |
| `sh` / `sk` | 前のペインへフォーカス移動 |
| `sl` / `sj` | 次のペインへフォーカス移動 |
| マウスクリック | クリックしたペインにフォーカスを移動 |
| `:q` | 現在のペインを閉じる（最後のペインならエディタ終了） |

## INSERT保存・文字/単語削除

| キー | 動作 |
|---|---|
| `Ctrl+]` / `Ctrl+[` | NORMALモードへ戻りつつ即座にファイル保存 |
| `Ctrl+D` | カーソル位置の1文字を削除（Emacsの `delete-char` 相当） |
| `Ctrl+K` | カーソルから行末まで削除（Emacsの `kill-line` 相当） |
| `Ctrl+W` | カーソル直前の1単語を削除（Vim互換、行頭をまたがない） |

## エラー・警告の一覧表示（`F2`）

`F2` はエディタ全体に効くグローバルキーで、カーソル行のコンパイルエラー・警告をダイアログで一覧表示します。エラー・警告がない場合は「この行にエラー・警告はありません。」と表示されます。

## フォントサイズ調整

現在アクティブなペインのビットマップフォントのセル幅・高さを調整できます（グローバルキー）。

| キー | 動作 |
|---|---|
| `Ctrl+Shift+←` / `Ctrl+Shift+→` | セル幅を縮小/拡大 |
| `Ctrl+Shift+↑` / `Ctrl+Shift+↓` | セル高さを縮小/拡大 |

## ステータスバーのアニメーション・時計

アクティブなペインのステータスバーには、24時間形式の時計（`HH:mm:ss`）と、歩くキャラクターのスプライトアニメーションが表示されます。ペイン分割時はフォーカス中のペインにのみ表示されます。オン/オフを切り替える設定コマンドは現状ありません。

## 作業ディレクトリの変更手段について

GUIメニューバー（File/Edit等）は存在しません。作業ディレクトリの変更やファイル操作はすべてキーボード操作またはCOMMANDモードのコマンド（`:cd`・`:e` 等）から行います。詳細は [検索・ナビゲーション](03-search-and-navigation.md) を参照してください。
