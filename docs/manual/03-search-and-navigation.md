# 3. 検索・ナビゲーション

## バッファ内文字列検索（SEARCHモード）

NORMALモードで `/` を押すとSEARCHモードに入り、画面下部に `/` に続けて入力中のパターンがリアルタイム表示されます。

| キー | 動作 |
|---|---|
| 通常文字 | 検索パターンに文字を追加 |
| `Backspace` | パターンの末尾の文字を削除 |
| `Enter` | 正規表現（`java.util.regex.Pattern`）として前方検索を実行しNORMALモードへ戻る。全マッチを半透明黄色でハイライトし、ステータスバーに `/pattern [N/M]` 形式でマッチ位置を表示 |
| `Escape` | 検索をキャンセルしてNORMALモードへ戻る（ハイライトもクリア） |

検索実行後、NORMALモードで以下のキーが使えます。

| キー | 動作 |
|---|---|
| `n` | 最後の検索と同方向で次のマッチへジャンプ（末尾まで達すると先頭へ折り返す） |
| `N` | 最後の検索と逆方向で前のマッチへジャンプ（折り返しあり） |
| `*` | カーソル位置の単語を単語境界（`\b`）付きで下方向へ完全一致検索 |
| `#` | カーソル位置の単語を単語境界付きで上方向へ完全一致検索 |

不正な正規表現を入力した場合は `E: bad pattern: ...` がステータスバーに表示されます。検索ハイライトはEscキーまたはファイル読み込み時にクリアされます（バッファ編集時には自動更新されないため、`n` を押すことで再計算されます）。

## ファイル名検索・ファイル内容grep（FILESEARCHモード）

| キー | 動作 |
|---|---|
| `\f` | FILESEARCHモードへ（ファイル名検索）。プロジェクト配下のファイル名を正規表現（大文字小文字無視）で検索 |
| `\g` | FILESEARCHモードへ（ファイル内容grep検索）。プロジェクト配下のファイル内容を正規表現でgrep |

FILESEARCHモード中の操作:

| キー | 動作 |
|---|---|
| 通常文字 | 検索パターンに文字を追加 |
| `Backspace` | パターンの末尾の文字を削除 |
| `Enter` | パターンで検索を実行しNORMALモードへ戻る（結果は疑似バッファに一覧表示、`Enter` でジャンプ） |
| `Escape` | 検索をキャンセルしてNORMALモードへ戻る |

### bang（`!`）付き全ファイル検索

デフォルトの検索は `.git` / `build` / `target` / `.gradle` / `node_modules` / `.idea` / `.vscode` をスキップします。パターンの**先頭に `!` を付けて**Enterすると、これらのスキップを無視して作業ディレクトリ配下の全ファイルを対象に検索します（`!` はパターンから取り除かれます）。

```
\f!pattern    → 全ファイル対象のファイル名検索
\g!pattern    → 全ファイル対象のgrep検索
```

`gR`（後述）・`:grep!`（後述）も同じ「bang = 全ファイルスキャン」の意味を持ちます。タイムアウト（後述）とファイルサイズ上限は bang の有無に関わらず常に適用されます。

## プロジェクト全文検索（`:grep`）

COMMANDモードで `:grep <pattern>` を実行すると、作業ディレクトリ配下を正規表現で全文検索し、結果を `*grep*` 疑似バッファに一覧表示します。

| コマンド | 動作 |
|---|---|
| `:grep <pattern>` | 通常の全文検索（スキップ対象ディレクトリを除外） |
| `:grep! <pattern>` | bang付き全文検索（スキップ対象ディレクトリも含めて全ファイル検索） |

`*grep*` 疑似バッファ内では以下のキーが使えます。

| キー | 動作 |
|---|---|
| `j` / `k` | 結果行を移動 |
| `Enter` | カーソル行の結果ファイルを開き、該当行へジャンプ |
| `:e <path>` | 別のファイルを開き、grepモードを解除する |

不正な正規表現には `E: bad pattern: ...`、一致なしには `grep: no matches for /<pattern>/` を表示します。2MBを超えるファイルは全文読み込みをスキップします（巨大ファイルによるフリーズ対策）。

## 参照検索（`gr` / `gR`）とシンボル定義ジャンプ（`K` / `Shift+J`）

| キー | 動作 |
|---|---|
| `gr`（`g` の後に `r`） | カーソル位置の識別子を単語境界付きでプロジェクト全体からgrep検索し、`*grep*` 疑似バッファに一覧表示（スキップ対象ディレクトリは除外） |
| `gR`（`g` の後に `Shift+R`） | `gr` のbang版。スキップ対象ディレクトリも含めて全ファイルを検索 |

`K`（定義・ドキュメント参照ジャンプ）と `Shift+J`（ジャンプ元へ戻る）は、JDKクラス参照だけでなくプロジェクト内シンボルの定義ジャンプも兼ねます。詳細は [Java開発支援](04-java-tooling.md#定義ジャンプとドキュメント参照-k--shiftj) を参照してください。

## telescope ファジーファインダー

telescope.vim風の3ペインオーバーレイUI（Prompt / Results / Preview）が画面中央に表示されます。

| キー | 起動する機能 |
|---|---|
| `SPC+f` | ファジーファイル検索 |
| `SPC+/` | ライブgrep |
| `SPC+b` | 開いているバッファの一覧 |

TELESCOPEモード中の操作:

| キー | 動作 |
|---|---|
| 通常文字 | クエリバッファに追加し、リアルタイムでフィルタリング |
| `Backspace` | クエリの末尾の文字を削除 |
| `Ctrl+N` / `Ctrl+P` | 候補リストを次/前へ移動 |
| `Enter` | 選択中の候補を開いてNORMALモードへ戻る（grep結果は該当行へジャンプ） |
| `Escape` | キャンセルしてNORMALモードへ戻る |

ファジーマッチは連続一致+3・単語境界+2・通常一致+1・ギャップ-1のスコアリングにより降順で表示されます。

## 作業ディレクトリと `:cd`

`:grep`・`:rename`・ファイル名検索・telescope・FILERはすべて `WorkingDirectoryManager` が管理する単一の作業ディレクトリを基準に動作します。

| コマンド | 動作 |
|---|---|
| `:pwd` | 現在の作業ディレクトリをステータスバーに表示 |
| `:cd <path>` | 作業ディレクトリを変更（相対パスは現在の作業ディレクトリから解決、`~` 展開に対応）。成功すると自動的にFILERモードへ遷移する |

作業ディレクトリを変更するGUIメニューは存在しません（`:cd` コマンドのみがこのエディタでの変更手段です）。

### `:cd`/`:e` のTABパス補完

COMMANDモードでコマンド入力中に `cd`/`cd <partial-path>` または `e`/`e <partial-path>` と入力した状態で `Tab` を押すと、パス名の前方一致補完が行われます。

- 候補が0件: 何もしない
- 候補が1件: その場で `commandBuffer` を補完（`:cd` はディレクトリ名の末尾に `/` を付与し、続けてTabで深掘り可能）
- 候補が2件以上: `*cd候補*` / `*e候補*` 疑似バッファに候補一覧を表示（`:e` はファイルとディレクトリの両方を候補にし、ディレクトリ名の末尾に `/` を付与）

疑似バッファ内では `Enter` で選択・補完を確定してCOMMANDモードの入力を再開し、`q` でキャンセルして元のバッファと入力途中のコマンド文字列を復元します。

## FILERモード（ディレクトリブラウザ）

`:cd <path>` が成功すると自動的にFILERモードへ遷移し、新しい作業ディレクトリの内容（ディレクトリ優先・アルファベット順）を3ペインオーバーレイで表示します（プレビューペインには選択項目の子要素やテキストプレビューが表示されます）。

| キー | 動作 |
|---|---|
| `Ctrl+N` / `Ctrl+P` | 選択項目を下/上へ移動 |
| `Enter` | ディレクトリを選択した場合はそのディレクトリへ移動して一覧を更新（ドリルダウン）。ファイルを選択した場合はそのファイルを開いてNORMALモードへ戻る |
| `/` | FILER内検索サブモードへ入る |
| `Esc` | FILERモードを終了してNORMALモードへ戻る |

FILER内検索サブモード（`/` 押下後）:

| キー | 動作 |
|---|---|
| 通常文字 | クエリに追加し、大文字小文字無視の部分一致でリアルタイムにフィルタリング |
| `Backspace` | クエリを1文字削除して再フィルタリング |
| `Ctrl+N` / `Ctrl+P` | フィルタ後の候補を移動 |
| `Enter` | 選択（ディレクトリ移動 or ファイルを開く） |
| `Esc` | 検索サブモードのみをキャンセルし、フィルタなしの一覧に戻る（FILERモード自体は継続） |

> 上位ディレクトリ（`..`）へのショートカットキーはありません。上に戻るには `Esc` でFILERを抜けてから、改めて `:cd ..` を実行してください。

## バッファ履歴の切り替え（`Ctrl+U` / `Ctrl+P`）

NORMALモードの `Ctrl+U` は「直前に開いたバッファへ切り替え」、`Ctrl+P` は「次のバッファへ切り替え」に割り当てられています（バッファ履歴のスナップショット方式）。`:tutor` でチュートリアルを開いた後に元のバッファへ戻る場合などに使えます。

## 検索処理のタイムアウトについて

`K`（[Java開発支援](04-java-tooling.md)参照）・`gr`/`gR`・`:grep`/`:grep!` によるプロジェクト全体検索は、対象ファイル数が多い作業ディレクトリ（例: ホームディレクトリ全体）で無制限にUIをフリーズさせないよう、最大1.5秒のタイムアウトが設定されています。タイムアウトした場合は検索を打ち切り、`K` はJDK側のフォールバック検索に切り替わり、`gr`/`:grep` は "search timed out" を表示します。
